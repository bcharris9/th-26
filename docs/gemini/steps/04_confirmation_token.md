# Confirmation Token Step 04

## Gemini Prompt
Please provide a confirmation token approach that is easy to explain to judges.

Ask for:
1) A simple token design you can explain to judges
2) What fields to bind: sessionId, action kind, payee or biller id, amount
3) Server-only validation approach
4) Failure cases to test

Output format:
- token design description
- validation rules list
- 6 test cases
